# 模型设定说明：情境是否影响行为、动机与阻碍的刻画、两行为之间路径

## 一、模型一：情境是否影响行为？是否需要“情境→行为”的间接？

### 当前设定

- **情境→行为 的间接路径**：有，且是模型一的核心。  
  情境（SMI, PSR, CTA）不直接解释行为，而是通过**动机层（EEM, GBI, RSA）**再指向**行为层（PVI, TWI）**，即：
  - 情境 → 动机 → 观演意愿（PVI）
  - 情境 → 动机 → 旅游消费意愿（TWI）  
  这就是“**情境通过动机间接影响行为**”的完整链条。

- **情境→行为 的直接路径**：目前只设了一条——**CTA → TWI**（城市文旅吸引力直接→旅游消费意愿），符合“为一场演出赴一座城”的文旅逻辑。

### 是否需要保留“情境→行为”的间接？

**需要，且应作为主路径。**  
理由：

1. **理论**：情境-动机-行为三层架构的含义就是“情境先激活动机，动机再驱动行为”；若没有情境→动机→行为这条间接路径，就退化成“情境直接预测行为”，和普通回归没区别，也失去“动机中介”的检验意义。
2. **实证**：间接效应 = 情境→动机的路径系数 × 动机→行为的路径系数，可以做中介分析（如 Bootstrap 置信区间）；若删掉间接，就做不了“动机是否中介情境对行为的影响”。
3. **与文档一致**：你的《结构方程模型》里方案一就是“情境层→动机层→行为层”，间接路径是题中应有之义。

**小结**：  
- 情境影响行为 = **间接（情境→动机→意愿）** + **直接（仅 CTA→TWI）**。  
- “情境影响行为”的**间接**必须保留；是否再增加其他情境→行为的直接路径（如 SMI→PVI）可按理论决定，当前只保留 CTA→TWI 是合理且常见的做法。

## 二、模型二：动机与阻碍的路径是怎么刻画的？

### 当前刻画

- **动机（MOT）**：二阶因子，由 EEM、GBI、RSA 反映；MOT 作为潜变量进入结构模型。
- **阻碍（PCB）**：一阶潜变量，独立进入结构模型。
- **结构路径**（四条）：
  - MOT → PVI（内在动机 → 观演意愿，正）
  - MOT → TWI（内在动机 → 旅游消费意愿，正）
  - PCB → PVI（感知成本障碍 → 观演意愿，负）
  - PCB → TWI（感知成本障碍 → 旅游消费意愿，负）

即：**动机与阻碍是平行、并列的预测变量**，共同解释 PVI 和 TWI，没有“动机→阻碍”或“阻碍→动机”的路径。  
含义是：  
- 动机越强 → 意愿越高（推）；  
- 阻碍越强 → 意愿越低（拉）；  
- 两者同时作用，形成“动机-阻碍博弈”对行为意愿的影响。

若你希望强调“动机与阻碍的交互”或“阻碍调节动机对意愿的作用”，可以在后续扩展中加交互项或分组分析；当前设定是**主效应平行**，清晰且易解释。

## 三、两个行为（PVI 与 TWI）之间是什么路径？是否应是“参与影响消费”？

### 当前设定

- 模型二中 **PVI（观演/参与意愿）** 和 **TWI（旅游/消费意愿）** 是**并列内生变量**，只由 MOT 和 PCB 预测，**二者之间没有路径**，即没有“PVI → TWI”。

### 是否应该是“参与影响消费”？

**从逻辑上，“参与意愿 → 消费意愿”很合理**：  
先有“要不要去现场”（PVI），再在“去”的前提下才有“花多少、是否延伸旅游消费”（TWI），即参与意愿可以**驱动**消费意愿。

因此，若你接受“参与影响消费”的设定，建议在模型二中**增加路径 PVI → TWI**，即：

- **结构模型**改为：
  - PVI ~ MOT + PCB
  - TWI ~ MOT + PCB **+ PVI**

这样：
- 动机、阻碍仍直接作用于 PVI 和 TWI；
- 同时 TWI 还受 PVI 影响：参与意愿越高，消费意愿越高（在控制 MOT、PCB 后）。

**小结**：  
- 当前：两个行为之间**没有**路径，不是“参与影响消费”。  
- 建议：若理论上认同“先参与、再消费”，可增加 **PVI → TWI**，模型二即变为“动机+阻碍→参与意愿→消费意愿”的部分链式结构。

## 四、若在模型二中增加 PVI→TWI，需要改的代码

- **sem_analysis.py**：`MODEL2_DESC` 中结构部分改为  
  `PVI ~ MOT + PCB`  
  `TWI ~ MOT + PCB + PVI`  
  并在 `MODEL2_HYPOTHESES` 中增加一条：PVI → TWI（参与意愿 → 旅游消费意愿，预期正）。
- **sem_regression_fallback.py**：TWI 的回归方程中增加自变量 PVI，即 TWI ~ MOT + PCB + PVI。
- **sem_charts.py**（若画图）：在模型二路径图中增加从 PVI 指向 TWI 的箭头并标系数。
