# 当前结果分析 + Prompt 分类数量与随机扰动说明

## 一、目前结果总览

### 1.1 数据与样本

- **来源**：synthetic_survey_data.csv，约 **N=200**，由 data_pipeline.py + DeepSeek API 生成。
- **画像**：由先验 P(E) 与条件概率 P(I|E) 依序采样得到，联合分布为 P(E,I)=P(E)×P(I|E)；再以人设 prompt 驱动 LLM 填答，逻辑连贯、非真实受访者。

### 1.2 信效度（9 个构念）

| 项目 | 结论 |
|------|------|
| **信度/收敛效度** | 8/9 达标（α、CR、AVE）；**EEM（情感体验动机）** 不达标（α=0.43，AVE=0.46），三题内部一致性弱。 |
| **区分效度** | 方案一 3 处违反（SMI-PSR、SMI-RSA、GBI-RSA），方案二 1 处违反（GBI-RSA）；动机构念间相关偏高（0.86～0.94）。 |
| **方向** | PCB 与动机/意愿负相关（约 -0.44～-0.83），符合“成本障碍”理论。 |

### 1.3 路径回归（两模型，近似 SEM）

- **模型一**：情境→动机→意愿 + CTA→TWI 直接 + PVI→TWI。  
  - 情境层：PSR 对 EEM、RSA 强显著；SMI 对 GBI 强显著；CTA 对 EEM、RSA、TWI 显著。  
  - 动机→意愿：RSA 对 PVI、TWI 贡献大；EEM 对 PVI 显著；GBI 多不显著（共线性）。  
- **模型二**：MOT、PCB 平行预测 PVI/TWI，PVI→TWI。  
  - MOT→PVI 强正、PCB→PVI 负；MOT→TWI 在加入 PVI 后变弱，PCB→TWI 强负；PVI→TWI 边缘显著（β≈0.19，p≈0.08）。

### 1.4 小结

- **优点**：量表结构基本成立（除 EEM）、路径方向符合理论、可复现流程与代码。  
- **局限**：EEM 信度不足；动机构念相关过高、区分效度有违反；结论仅适用于当前合成数据，不能替代真实问卷。

## 二、Prompt 里“分类数量”是少还是多？

### 2.1 当前各维度分类数

| 维度 | 类别数 | 具体 |
|------|--------|------|
| 性别 | 2 | 女 / 男 |
| 年龄职业 | 3 | 17-21 学生 / 22-26 职场新人 / 27-31 资深职场 |
| 收入 | 5 | 5 档 |
| 常住地 | 3 | 天津本地 / 京津冀近途 / 远途 |
| 社交驱动 | 3 | 饭圈打卡 / 精致出片 / 解压放松 |
| 消费偏好 | 3 | 抠门陪伴党 / 理性消费者 / 极致前排控 |
| 互动执念 | 2 | 强 / 弱 |
| 文旅精力 | 3 | 充沛 / 一般 / 匮乏 |

理论上的**画像组合数** = 2×3×5×3 ×（条件采样的心理组合）≈ 数千级，所以**总体“类型”并不少**。

### 2.2 是“少”还是“多”？

- **不算少**  
  - 每个维度 2～5 类，在行为/营销类合成数据里属于常见粒度。  
  - 组合数已足够产生丰富差异，当前信效度与路径结果也说明数据有变异（除 EEM 外）。  
  - 再**大幅增加**类别（如消费偏好拆成 5～7 档）：  
    - 条件概率表会变复杂、难维护、难给依据；  
    - Prompt 更长、更碎，LLM 对过细标签的区分度有限，收益不一定大。

- **也不算过多**  
  - 若再**减少**（如消费偏好只留 2 类）：  
    - 行为差异会被压扁，方差变小，路径/信效度更容易“过于整齐”或失真。  
  - 当前 2～5 类/维度的设计在“可解释性”和“多样性”之间比较平衡。

**结论**：  
- **当前分类数量是适中的**，不必为“怕少”而整体加很多类。  
- 若希望**略微增加多样性**，可只在 1～2 个维度上微调，例如：  
  - 消费偏好：在“理性”和“极致”之间加一档“适度消费型”（共 4 档）；或  
  - 社交驱动：保持 3 类，但把描述写得更区分一些。  
- **不建议**：每个维度都往 5～7 类扩，容易过细、难辩护、prompt 负担大。

## 三、需不需要加入随机扰动？

### 3.1 为什么要考虑扰动？

- 同一类人设（如“女 / 22-26 职场 / 理性消费者”）的 prompt 表述若**完全固定**，不同条数据可能得到**非常相似**的作答，导致：  
  - 同一“类型”内方差偏小；  
  - 分布过于“整齐”，更像按类型生成而非真人波动。  
- 适度扰动可以：  
  - 减轻**模板感**；  
  - 让同一类人在表述上略有差异，从而**增加作答多样性**，更接近真实问卷的噪声。

### 3.2 建议：**适度加入扰动**，但要有边界

推荐做、且易实现的两种：

1. **表述扰动（推荐）**  
   - **做法**：同一类人设用**多套等价说法**随机选一，例如：  
     - “理性消费者” → 有时写 “会权衡性价比的消费者”、“消费上比较理性的观众”；  
     - “精力充沛” → 有时写 “出行意愿很强，愿意为演出多花时间精力”。  
   - **目的**：不改变人设含义，只改变 wording，减少固定模板、拉高作答差异。  
   - **注意**：不要改到语义不一致（如把“抠门”说成“舍得花钱”）。

2. **采样权重扰动（可选）**  
   - **做法**：在抽样时对先验/条件概率加**小随机扰动**，例如权重乘以 (1 + ε)，ε 从 N(0, 0.05) 或 Uniform(-0.05, 0.05) 抽，再归一化。  
   - **目的**：每批生成的数据边际分布不会**完全**等于你设定的 P(E)，更接近“从近似该分布的总体里抽样”。  
   - **注意**：扰动要小，避免某些类别概率被压到接近 0 或主导一切。

不建议做的：

- **对最终 1–5 题分或选项做数值/随机扰动**：容易破坏“人设→作答”的逻辑一致性，且难以解释，审稿也容易质疑。

### 3.3 小结

- **分类数量**：当前**适中**，不必整体加很多类；若要微调，可在 1～2 个维度略增（如消费偏好多一档）。  
- **随机扰动**：**建议加**，且优先加**表述扰动**（同一人设多种说法）；权重扰动可选、宜小。不对题项得分做事后扰动。
